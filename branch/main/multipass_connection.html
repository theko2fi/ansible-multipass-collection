theko2fi.multipass.multipass connection – Run tasks in multipass virtual machines

Note

This connection plugin is part of the theko2fi.multipass collection (version 1.0.0).

To install it, use: ansible-galaxy collection install theko2fi.multipass.

To use it in a playbook, specify: theko2fi.multipass.multipass.

  • Synopsis

  • Parameters

Synopsis

  • Run commands or put/fetch files to an existing multipass VM.

  • Uses the multipass CLI to execute commands in the virtual machine.

Parameters

Parameter

Comments

container_timeout

integer

Controls how long we can wait to access reading output from the VM once execution started.

Default: 10

Configuration:

  • INI entries:

    [defaults]
timeout = 10

    [multipass_connection]
timeout = 10

    added in theko2fi.multipass 2.2.0

  • Environment variable: ANSIBLE_TIMEOUT

  • Environment variable: ANSIBLE_MULTIPASS_TIMEOUT

    added in theko2fi.multipass 2.2.0

  • Variable: ansible_multipass_timeout

    added in theko2fi.multipass 2.2.0

  • CLI argument: –timeout

multipass_extra_args

string

Extra arguments to pass to the multipass command line.

Default: ""

Configuration:

  • INI entry:

    [multipass_connection]
extra_cli_args = ""

  • Variable: ansible_multipass_extra_args

remote_addr

string

The name of the VM you want to access.

Default: "inventory_hostname"

Configuration:

  • Variable: inventory_hostname

  • Variable: ansible_host

  • Variable: ansible_multipass_host

remote_user

string

The user to execute as inside the container.

If multipass is too old to allow this (< 1.7), the one set by multipass itself will be used.

Configuration:

  • INI entry:

    [defaults]
remote_user = VALUE

  • Environment variable: ANSIBLE_REMOTE_USER

  • Variable: ansible_user

  • Variable: ansible_multipass_user

  • Keyword: remote_user

  • CLI argument: –user

Authors

  • Kenneth KOFFI (@theko2fi)

Hint

Configuration entries for each entry type have a low to high priority order. For example, a variable that is lower in the list will override a variable that is higher up.
